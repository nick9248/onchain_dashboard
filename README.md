# Onchain Dashboard

A local web dashboard for viewing on-chain options analytics reports for BTC and ETH.

## What it shows

- GEX and DEX by strike
- IV surface and vol surface analysis
- Max pain and distance from current price
- Put/Call ratio by strike and moneyness
- Buying and selling pressure by strike
- Block trades (size, direction, notional, IV)
- Synthesis summary: regime, direction, vol assessment, risk factors, trade recommendations

## Architecture

```
onchain_dashboard/
├── dashboard/
│   └── dashboard_server.py   # Flask API + static file server
└── dashboard/frontend/       # React + Vite frontend
    ├── src/
    └── dist/                 # Built output served by Flask
```

The Flask server exposes one API endpoint (`/api/data`) that reads the latest
report files for BTC and ETH, parses them, and returns structured JSON. It also
serves the built React app as static files.

## Setup

### Backend

```bash
pip install flask flask-cors
python dashboard/dashboard_server.py
```

The server starts on `http://127.0.0.1:5000` and opens a browser tab automatically.

### Frontend

Build the React app once (or after any frontend changes):

```bash
cd dashboard/frontend
npm install
npm run build
```

The built files go to `dashboard/frontend/dist/`, which Flask serves directly.

## Configuration

Set the `ONCHAIN_DATA_PATH` environment variable to point to the directory
containing per-asset report folders:

```bash
# Windows
set ONCHAIN_DATA_PATH=C:\path\to\onchain_analysis

# macOS / Linux
export ONCHAIN_DATA_PATH=/path/to/onchain_analysis
```

If unset, defaults to `../data/onchain_analysis` relative to `dashboard_server.py`.

## Data format

The server expects this directory structure:

```
{ONCHAIN_DATA_PATH}/
├── BTC/
│   └── report/
│       └── 20250501_120000/   # timestamped subdirectory (latest is used)
│           ├── report.txt
│           └── synthesis.txt
└── ETH/
    └── report/
        └── 20250501_120000/
            ├── report.txt
            └── synthesis.txt
```

Reports are generated by the companion `option_trading` project.
